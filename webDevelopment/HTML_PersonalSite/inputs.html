Input type Actual input Short explanation
input type="button" name="" value="button" A push button with no default behavior displaying the value of the value attribute, empty by default.
input type="text" name="" value="" A check box allowing single values to be selected/deselected.
input type="color" name="" value="" A control for specifying a color; opening a color picker when active in supporting browsers.
input type="date" name="" value="" A control for entering a date (year, month, and day, with no time). Opens a date picker or numeric wheels for year, month, day when active in supporting browsers.
input type="datetime-local" name="" value="" A control for entering a date and time, with no time zone. Opens a date picker or numeric wheels for date- and time-components when active in supporting browsers.
input type="email" name="" value="" A field for editing an email address. Looks like a text input, but has validation parameters and relevant keyboard in supporting browsers and devices with dynamic keyboards.
input type="file" name="" value="" A control that lets the user select a file. Use the accept attribute to define the types of files that the control can select.
input type="hidden" name="" value="" A control that is not displayed but whose value is submitted to the server. There is an example in the next column, but it's hidden!
input type="image" name="" value="" A graphical submit button. Displays an image defined by the src attribute. The alt attribute displays if the image src is missing.
input type="month" name="" value="" A control for entering a month and year, with no time zone.
input type="number" name="" value="" A control for entering a number. Displays a spinner and adds default validation when supported. Displays a numeric keypad in some devices with dynamic keypads.
input type="password" name="" value="" A single-line text field whose value is obscured. Will alert user if site is not secure.
input type="radio" name="" value="" A radio button, allowing a single value to be selected out of multiple choices with the same name value.
input type="range" name="" value="" A control for entering a number whose exact value is not important. Displays as a range widget defaulting to the middle value. Used in conjunction min and max to define the range of acceptable values.
input type="reset" name="" value="reset" A button that resets the contents of the form to default values. Not recommended.
input type="search" name="" value="" A single-line text field for entering search strings. Line-breaks are automatically removed from the input value. May include a delete icon in supporting browsers that can be used to clear the field. Displays a search icon instead of enter key on some devices with dynamic keypads.
input type="submit" name="" value="submit" A button that submits the form.
input type="tel" name="" value="" A control for entering a telephone number. Displays a telephone keypad in some devices with dynamic keypads.
input type="text" name="" value="" The default value. A single-line text field. Line-breaks are automatically removed from the input value.
input type="time" name="" value="" A control for entering a time value with no time zone.
input type="url" name="" value="" A field for entering a URL. Looks like a text input, but has validation parameters and relevant keyboard in supporting browsers and devices with dynamic keyboards.
input type="week" name="" value="" A control for entering a date consisting of a week-year number and a week number with no time zone.
